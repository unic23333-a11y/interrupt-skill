# 插一句嘴

## 核心功能

当用户在AI正在执行任务时插入内容，Skill自动判断：
1. **是否和当前任务相关**
2. **是修改原任务还是新开任务**
3. **执行对应的处理逻辑**

---

## 触发条件

满足任一即触发：

| 条件 | 说明 |
|------|------|
| AI正在执行任务 | 生成中/思考中/搜索中/工具调用中 → 用户任何输入都触发 |
| 逗号开头 | 任何语言的逗号（`,` `，` `、`等）作为输入开头 |

---

## 判断流程

```
用户输入
    ↓
【判断1】是否触发？
    ├─ AI正在执行任务？→ yes → 进入判断2
    └─ 逗号开头？→ yes → 进入判断2
                └─ no → 正常对话
            ↓
      【判断2】和当前任务有关系？
        ├─ 无关 → 直接回答，继续当前任务
        └─ 有关 → 进入判断3
                ↓
          【判断3】是改原任务还是新开任务？
            ├─ 改/增补原任务
            │   → 直接修改/增补当前输出，继续生成
            │
            └─ 新开任务
                → 检查队列（≤3个）
                    ├─ 队列未满 → 后台挂起当前任务，开始新任务
                    └─ 队列已满 → 提示用户先完成一个
```

---

## 相关性判断规则

**强相关**（改/增补原任务）：
- 内容和当前任务的主题相关
- 内容是当前上下文的补充/修正
- 内容针对AI正在生成的内容

**完全无关**（直接回答，不打断）：
- 闲聊、天气、问候等
- 和当前任务毫无关联

---

## 任务队列规则

- **最大数量**：3个任务
- **队列已满时**：提示用户先完成一个任务
- **任务状态**：
  - `active` — 正在执行
  - `pending` — 后台挂起
  - `completed` — 已完成

---

## 上下文保留

- **活跃任务**：保留完整的已完成内容 + 当前生成进度
- **后台任务**：保留任务目标和已完成内容，清除生成进度
- **切换时**：直接恢复到任务状态，继续生成

---

## 指令解析

| 指令类型 | 示例 | 处理方式 |
|---------|------|---------|
| 修改内容 | "把Objective改成..." | 定位并修改对应部分 |
| 增补内容 | "再加一个作品集" | 在当前位置后增补 |
| 纠正方向 | "不对，我要的是..." | 更新上下文，重新生成 |
| 参数更新 | "把Python改成精通" | 更新参数，重新生成 |

## 使用限制

- 队列最多3个任务
- 修改原任务后会重新生成（不是原地编辑）
- 上下文切换会有轻微延迟（恢复后台任务状态）
